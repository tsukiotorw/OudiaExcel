## 第3回調査（2026-07-19）

### 調査対象

- 月音旅客鉄道.oud2
- OudiaSecond GUI にて確認

### 調査目的

EkiJikoku レコードの構造および各フィールドの意味を確認する。

---

### 1. stop_flag

EkiJikoku レコード先頭の数値は停車種別を表す。

|値|意味|
|---|---|
|1|停車|
|2|通過|

GUI上で「停車」「通過」の設定と一致することを確認した。

GUIには「運行なし」の設定も存在するが、この場合は stop_flag を持たず、空レコードとなる。

例

```
1;503/504$0
2;551$1
```

---

### 2. track

`$` の後ろの値は番線番号ではなく、EkiTrack2 のインデックスである。

対応先

```
Eki.
└── EkiTrack2Cont.
    ├── EkiTrack2 (0)
    ├── EkiTrack2 (1)
    ├── EkiTrack2 (2)
    ...
```

観測結果

- 0 ～ 12 を確認
- OudiaSecond の仕様上はさらに大きな値も設定可能と思われる

よって、この値は固定範囲ではなく EkiTrack2 配列への参照インデックスと考えられる。

---

### 3. 通過レコード

通過には二種類の表現が存在する。

#### 通過時刻あり

```
2;551$1
```

意味

- stop_flag = 2
- 通過時刻 = 5:51
- track = 1

#### 通過時刻なし

```
2$1
```

意味

- stop_flag = 2
- 通過時刻なし
- track = 1

したがって、

```
2$1
```

は特殊レコードではなく、

```
2;551$1
```

から通過時刻を省略した形式である。

---

### 4. 空レコード

空レコード

```
,
```

または

```
,,,,
```

は以下の両方のケースで使用される。

- 駅を通らない
- データなし（運行なし）

GUI上では両者は区別されるが、保存形式では区別されていない。
